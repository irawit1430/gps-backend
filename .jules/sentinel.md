## 2025-02-18 - Authorization Bypass on Administrative Endpoints
**Vulnerability:** Found multiple sensitive endpoints (like `/api/admin/stats`, `/api/admins`, and `/api/settings`) that lacked role-based authorization checking. Only basic authentication (`authenticate` middleware) was present, allowing any authenticated user (e.g. `PARENT` or `DRIVER`) to access administrative endpoints.
**Learning:** In a multi-role application, validating the JWT presence is insufficient. Failing to implement explicit authorization checks on high-value endpoints exposes them to Insecure Direct Object References (IDOR) and privilege escalation.
**Prevention:** Implement and enforce role-based access control (RBAC) middleware uniformly. Utilize a reusable `authorizeRoles(...roles)` middleware for sensitive routes to assert `req.user.role` prior to fulfilling the request.
